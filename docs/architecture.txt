│                  KYC Network Scout                       │
│            Архітектура (поточна)                          │
└──────────────────────────────────────────────────────────┘

ВХІД
   ┌─ Веб (FastAPI, :8000) ── форма + пакетна перевірка (до 5)
   │     POST /api/check → фоновий пул → polling → готовий звіт
   │     (basic-auth через KYC_WEB_PASSWORD; персист задач у output/.jobs)
   │
   └─ CLI ── docker compose run --rm kyc <ЄДРПОУ | РНОКПП | назва>
            │
            ▼
   ┌──────────────────────────────────────────────────────┐
   │  run_kyc (синхронний оркестратор)                    │
   │  input_resolver → тип суб'єкта + резолвинг назви      │
   └───────┬──────────────────────────────────────────────┘
           │  (послідовно, чесна деградація — збій джерела
           │   не валить прогін, лише позначається в покритті)
           ▼
   ┌─ Worker 1  registry_task / person_task  ── Opendatabot (реєстр, PEP, factors)
   ├─ Worker 2  sanctions_task ─────────────── Opendatabot (санкції; фізособи: list-complete)
   └─ Worker 3  run_adverse_media ──────────── Serper → SSRF-гард → Crawl4AI → Claude
           │       (пошук за брендом UA/EN; класифікатор відсіює однофамільців)
           ▼
   ┌──────────────────────────────────────────────────────┐
   │  aggregator                                          │
   │  entity resolution + скоринг ризику (0–100) +        │
   │  audit trail + граф (pyvis) + evidence ZIP           │
   │  → output/<timestamp>/: kyc_report.json, report.html,│
   │     graph.html, evidence.zip                         │
   └──────────────────────────────────────────────────────┘

Усе зшивається під єдиним slug (company_<ЄДРПОУ> / person_<РНОКПП>).

─────────────────────────────────────────────────────────────
ASYNC-РЕЖИМ (масштабування, опційно) — Celery + Redis
─────────────────────────────────────────────────────────────
   search_task (worker-search) → генерує запити → Serper → URLs
        │  apply_async() по кожному URL
        ▼
   Redis broker → scrape_and_classify_task * N (worker-scraper)
        │  Crawl4AI + Playwright → markdown → Claude → JSON
        ▼
   data/normalized/{slug}/{hash}.json
   Beat scheduler → heartbeat (health)

Синхронний run_kyc не потребує Redis — найпростіший шлях для одиничних
та невеликих пакетних перевірок; Celery-шлях лишається для масштабу.
